## 🔄 МОДЕЛЬ ПРОГРЕССИИ: непрерывная по возрасту и глубине

Программа не имеет фиксированного срока. Ребёнок растёт внутри неё, меняя трек при переходе из детского сада в школу.
На школьном треке каждый Z-принцип сопровождается FP-маркерами (версия->проверка, честное сравнение условий, согласование критерия).

### Треки MVP

```
ДОШКОЛЬНЫЙ ТРЕК (3–6 лет) — Глубина 1
──────────────────────────────────────────────
Стратегия: breadth-first (все принципы поочерёдно на @1)
Форма: тактильные эпизоды, бытовые ситуации, физические объекты

Рекомендуемый порядок:
Z2 → Z0 → Z3 → Z1 → Z5 → Z4 → Z6 → Z7
│         │
≈1 цикл   ≈1 цикл  (обычно ~1 месяц практики на принцип)

Карточки: 03-methods/preschool/


ШКОЛЬНЫЙ ТРЕК (7–10 лет) — Глубина 1–2
──────────────────────────────────────────────
Стратегия: breadth-first + углубление (Глубина 2)
Форма: вербальные задачи, объяснения, связь со школьными предметами

Рекомендуемый порядок:
Z2 → Z0 → Z3 → Z1 → Z5 → Z4 → Z6 → Z7
    (Глубина 1 ≈ 1 цикл (~1 мес) → Глубина 2 ≈ 1 цикл (~1 мес) → следующий принцип)

Карточки: 03-methods/school/


ПЕРЕХОДНАЯ ЗОНА (11–13 лет) — Глубина 2–3
──────────────────────────────────────────────
Вне MVP — разрабатывается отдельно.
```

### Цикл по принципу (операционная единица для обоих треков)

```
ЦИКЛ (один принцип Z{N}; обычно 3–6 недель, без жёсткой даты)
├─ Старт (первые 3–7 дней)
│  ├─ Взрослый читает карточку эпизода
│  ├─ 1–2 пробных эпизода
│  └─ Первые наблюдения → лог
│
├─ Основной ритм (остальные недели цикла)
│  ├─ На входе: 2–3 эпизода в неделю
│  ├─ По готовности: выходим к почти ежедневному ритму (4–6 эпизодов в неделю)
│  ├─ Меняем контекст (не только ванная → добавить кухню/прогулку)
│  └─ Смотрим: есть ли спонтанность?
│
└─ Завершение цикла (по гейтам переноса)
   ├─ Обновить Mastery Map (уровень 0–3 по этому принципу)
   ├─ Рефлексия взрослого
   └─ Guard Policy: переходим к следующему принципу?
      ✓ Уровень 3 зафиксирован ≥ 3 раза
      ✓ Перенос в ≥ 2 контекстах
      ✓ Спонтанность (без подсказки)
      ✓ Для 7–10: есть FP-перенос (FP-B/FP-C/FP-A1 или FP-F)
      → Да: следующий принцип
      → Нет: остаёмся ещё один цикл (меняем контекст, не принцип)
```

## ✅ ПРОЦЕССНЫЕ ГЕЙТЫ (контрольные точки)

| Гейт | Когда | Что проверить | Что дальше? |
|--------|------|---|---|
| **День 1** | Сразу | Согласованность взрослых, есть ли место для логов | Вперёд / Пауза если рассинхрон |
| **Конец недели 1** | После 1–2 эпизодов | Получилось ли? Ребёнок был в ресурсе? | Вперёд / Упростить формат |
| **Конец недели 2** | После регулярности | Ребёнок готов? Видны ли первые признаки? | Вперёд / Менять контекст или взрослого |
| **Завершение цикла** | По гейтам переноса (часто 3–6 недель) | Guard Policy: Уровень 3 ≥ 3 раза? Перенос в 2+ контекстах? | Следующий принцип / Ещё один цикл |
| **Переход трека** | При смене дошк.→школа | Форма подачи изменилась? Карточки из school/? | Перейти на school/ карточки |

## 📊 КОМПОНЕНТЫ И ИХ ВЗАИМОДЕЙСТВИЕ

### Компонент 1: РАЗЛИЧЕНИЕ (из 01-contract)
**Функция:** Точное описание что это такое  
**Входит в:** Методологию, логи, карты  
**Выход:** Ребёнок применяет в жизни

### Компонент 2: РОЛЬ (из 02-entities)
**Функция:** Кто что делает  
**Входит в:** Явный контракт между взрослыми  
**Выход:** Согласованная работа

### Компонент 3: МЕТОД (из 03-methods)
**Функция:** Пошаговый протокол эпизода  
**Входит в:** Каждый эпизод с ребёнком  
**Выход:** Регулярные, качественные эпизоды

### Компонент 4: ЛОГ (из 04-products)
**Функция:** Фиксация наблюдений  
**Входит в:** Каждый эпизод (3 строки) + обзор цикла  
**Выход:** Данные для корректировки

### Компонент 5: КАРТА (Mastery Map)
**Функция:** Видеть прогресс по 8 Z-принципам  
**Входит в:** Обзор цикла  
**Выход:** Решение, что делать дальше

### Компонент 6: ЗАЩИТА (из 05-failure)
**Функция:** Вектор от типовых ошибок  
**Входит в:** Осознанность взрослого  
**Выход:** Свободнее от иллюзий, четче решения

## 🔗 КАК PACK "СИДИТ" В БОЛЬШУЮ СИСТЕМУ

```
ZP (Нулевые принципы)         ← Фундамент
        ↑
       FPF (Первые принципы + фреймворк)  ← Основание
        ↑
        SPF (Форма для вторых принципов)  ← Оболочка
        ↑
   KIDS-PACK (Вторые принципы для детей) ← МЫ ЗДЕСЬ
        ↑
   Конкретная семья (3-й принципы)       ← Применение
```

**Логика:**
- Если ZP нарушаю → пересматриваю основание (FPF)
- Если FPF не подходит → пересматриваю форму (SPF)
- Если SPF не работает → пересматриваю содержание (KIDS-PACK)
- Если KIDS-PACK не работает → адаптирую для своей семьи

## 📈 ОЖИДАЕМАЯ ТРАЕКТОРИЯ (типичный случай, один принцип)

```
Первые 1–2 недели цикла (один принцип):
- Взрослый учится держать различение
- Ребёнок начинает замечать в конкретных ситуациях
- Уровень: 0–1 (с подсказкой)

Следующие 1–3 недели цикла (тот же принцип):
- Взрослый ведёт без скрипта
- Ребёнок связывает вопрос, проверку, ответ
- Уровень: 1–2 (самостоятельно в знакомом контексте)

После Guard Policy (60–80% спонтанности):
- Ребёнок применяет принцип без подсказки в новых ситуациях
- Уровень: 3 (перенос) → переходим к следующему принципу

По всем 8 принципам (Z0–Z7):
Дошкольный трек (3–6): первый круг @1 обычно ~8–12 месяцев
Школьный трек (7–10): круг @1→@2 обычно ~8–16 месяцев
Далее: следующий круг на большей глубине
```
